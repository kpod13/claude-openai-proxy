## MODIFIED Requirements

### Requirement: autorun install provisions user-level autostart
`autorun install` SHALL register the proxy binary as a user-level autostart entry that runs after the current user logs in. The mechanism used SHALL be appropriate for the host OS:
- **macOS**: launchd user agent plist at `~/Library/LaunchAgents/com.claude-openai-proxy.plist`, activated with `launchctl bootstrap gui/<uid>`. The plist SHALL run the proxy as a keep-alive agent (`KeepAlive` enabled) and SHALL set `EnvironmentVariables` `PATH` to include the common CLI locations `/opt/homebrew/bin`, `/usr/local/bin`, `/usr/bin`, and `/bin` so the launched server can locate the `claude` CLI.
- **Linux**: systemd user unit at `~/.config/systemd/user/claude-openai-proxy.service`, enabled with `systemctl --user enable --now`. Falls back to XDG autostart (`~/.config/autostart/claude-openai-proxy.desktop`) if systemd is unavailable.
- **Windows**: registry value under `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` with key `claude-openai-proxy`.

The entry SHALL use the absolute path of the currently running binary (`os.Executable()`).
The install command SHALL NOT require root or administrator privileges.

#### Scenario: Install on macOS creates plist and bootstraps agent
- **WHEN** `autorun install` is run on macOS
- **THEN** `~/Library/LaunchAgents/com.claude-openai-proxy.plist` is created and `launchctl bootstrap gui/<uid>` is executed

#### Scenario: macOS plist provides a working PATH and keep-alive
- **WHEN** the macOS plist is generated by `autorun install`
- **THEN** it sets `KeepAlive` enabled and an `EnvironmentVariables` `PATH` containing `/opt/homebrew/bin`, `/usr/local/bin`, `/usr/bin`, and `/bin`

#### Scenario: Install on Linux creates systemd user unit
- **WHEN** `autorun install` is run on Linux with systemd available
- **THEN** `~/.config/systemd/user/claude-openai-proxy.service` is created and `systemctl --user enable --now` is executed

#### Scenario: Install on Windows writes registry key
- **WHEN** `autorun install` is run on Windows
- **THEN** the registry key `HKCU\Software\Microsoft\Windows\CurrentVersion\Run\claude-openai-proxy` is set to the binary path

### Requirement: autorun uninstall removes the autostart entry
`autorun uninstall` SHALL remove the autostart entry created by `autorun install`, using the same OS-specific mechanism. It SHALL NOT fail if no entry exists (idempotent).

#### Scenario: Uninstall on macOS boots out and removes plist
- **WHEN** `autorun uninstall` is run on macOS and the plist exists
- **THEN** `launchctl bootout gui/<uid>` is executed and the plist file is deleted

#### Scenario: Uninstall on Linux disables and removes unit
- **WHEN** `autorun uninstall` is run on Linux and the unit exists
- **THEN** `systemctl --user disable --now` is executed and the unit file is deleted

#### Scenario: Uninstall on Windows removes registry key
- **WHEN** `autorun uninstall` is run on Windows
- **THEN** the `claude-openai-proxy` key is removed from `HKCU\...\Run`

#### Scenario: Uninstall is idempotent
- **WHEN** `autorun uninstall` is run and no entry exists
- **THEN** the command exits successfully without error

## ADDED Requirements

### Requirement: macOS plist is well-formed XML
The generated macOS LaunchAgent plist SHALL be well-formed XML. In particular, the leading XML declaration (`<?xml version="1.0" encoding="UTF-8"?>`) SHALL be emitted verbatim and not HTML-escaped, while user-controlled values (binary path, label) SHALL be XML-escaped.

#### Scenario: XML declaration is not escaped
- **WHEN** the macOS plist is generated by `autorun install`
- **THEN** it begins with `<?xml version="1.0" encoding="UTF-8"?>` and contains no `&lt;?xml` sequence

#### Scenario: Special characters in the binary path are escaped
- **WHEN** the binary path contains XML special characters such as `&`
- **THEN** they are emitted as XML entities (e.g. `&amp;`) so the plist remains well-formed

### Requirement: macOS install is atomic on activation failure
On macOS, if `launchctl bootstrap` fails, `autorun install` SHALL remove the plist file it just wrote before returning the error, leaving no partial autostart state behind.

#### Scenario: Bootstrap failure removes the written plist
- **WHEN** `autorun install` writes the plist but `launchctl bootstrap` returns an error
- **THEN** the command exits non-zero AND `~/Library/LaunchAgents/com.claude-openai-proxy.plist` does not remain on disk
